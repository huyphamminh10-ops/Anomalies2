​Mục tiêu: Chỉnh sửa file server.py (Dashboard) và config_manager.py (Bot) để đảm bảo dữ liệu đồng bộ 100%, không bị lỗi 500 trên Render và lưu trữ đúng theo yêu cầu (MongoDB cho cấu hình, TiDB cho Log/Ban/Feedback).
​Yêu cầu kỹ thuật chi tiết:
​1. Đồng bộ hóa Kiểu dữ liệu (ID Stringify Rule)
​Vấn đề: MongoDB phân biệt 123 (int) và "123" (string). Bot dùng string, Dashboard đôi khi dùng int.
​Nhiệm vụ:
​Trong tất cả các hàm xử lý tại server.py và config_manager.py, mọi biến guild_id, user_id, channel_id phải được ép kiểu qua str() trước khi thực hiện find, update_one hoặc insert.
​Sửa lại các filter truy vấn từ {"guild_id": {"$in": [int(id), str(id)]}} thành {"guild_id": str(id)} duy nhất để tránh xung đột index.
​2. Tối ưu kết nối MongoDB trên Render (Fix lỗi 500/Timeout)
​Nhiệm vụ: Cập nhật hàm get_db() hoặc khởi tạo MongoClient.
​Thêm serverSelectionTimeoutMS=5000 và connectTimeoutMS=10000.
​Thêm cơ chế kiểm tra kết nối (client.admin.command('ping')) trước khi trả về đối tượng database. Nếu lỗi, trả về None và log lỗi rõ ràng thay vì để sập cả tiến trình.
​3. Tích hợp TiDB cho Feedback, Ban và Update Log
​Bảng Feedbacks: * Sử dụng ID là chuỗi 10 ký tự ngẫu nhiên (chữ và số).
​Lưu các cột: id (PK), user_id, content, images (dạng JSON string), reply, created_at.
​Bảng Changelogs: * Sử dụng ID kiểu INT AUTO_INCREMENT PRIMARY KEY (tự động tăng từ 1).
​Lưu các cột: id, version, title, content, created_at.
​Bảng Bans: Quản lý ID bị cấm, lưu trên TiDB thay vì MongoDB.
​4. Xử lý Logic Trạng thái Phòng chơi (Lobby State)
​Nhiệm vụ: * Khi Dashboard gọi API lấy danh sách server, phải thực hiện "Lookup" hoặc query bổ sung sang collection lobby_states (MongoDB) để lấy trạng thái is_playing và player_count thực tế mà Bot đang ghi nhận.
​Không được chỉ đọc ở guild_configs vì bảng này chỉ chứa cài đặt tĩnh.
​5. Cơ chế Invalidate Cache (Đồng bộ thời gian thực)
​Vấn đề: Bot thường cache config vào RAM. Dashboard sửa DB nhưng Bot không biết.
​Nhiệm vụ: * Xây dựng một API nội bộ hoặc một trường last_updated trong MongoDB.
​Trong Bot (config_manager.py), khi kiểm tra config, nếu last_updated trong DB mới hơn bản trong RAM thì phải buộc tải lại (Reload).
​File cần chỉnh sửa:
​server.py: Sửa toàn bộ API CRUD, thêm logic TiDB, sửa kết nối Mongo.
​config_manager.py: Đồng bộ lại hàm load/save, ép kiểu string cho ID.
​index.html: Cập nhật hàm gọi API gửi Feedback và hiển thị trạng thái phòng.
